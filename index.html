AWS CI/CD Pipeline Infrastructure

Jenkins | SonarQube | Docker

Server 1: Jenkins

  • Instance: t2.micro
  • Ports: 8080, 22
  • The "Manager" - Orchestrates the builds and pulls from GitHub.

Server 2: SonarQube

  • Instance: t2.medium
  • Ports: 9000, 22
  • The "Inspector" - Scans code for vulnerabilities and bugs.

Server 3: Docker

  • Instance: t2.micro
  • Ports: 80, 22
  • The "Builder" - Hosts the final containerized website.

Manual Authentication Strategy

SSH Key Exchange

Instead of using a PEM file, Jenkins connects to Docker via Public Key Authorization:

  1. Generate ssh-keygen on Jenkins.
  2. Add id_rsa.pub to Docker's authorized_keys.
  3. Add Jenkins Private Key to Publish Over SSH plugin.

SonarQube Integration

Secure token-based handshake between Manager and Inspector:

  • Generate Token on SonarQube Dashboard.
  • Store in Jenkins Credentials as Secret Text.
  • Configure Sonar Scanner in Jenkins System Tools.
